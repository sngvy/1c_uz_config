Документ "Загрузка задолженности"

 

После принятия в работу объектов системы (например, долговых обязательств) необходимо провести загрузку сумм задолженностей по каждому их данных долговых обязательств. Принцип заполнения данного документа аналогичен принципу заполнения документа «Загрузка долговых обязательств»:

·       выгрузка необходимой информации из программного продукта, в котором ведётся бухгалтерский учёт сумм задолженностей по долговым обязательствам контрагентов, в Excel-таблицу;

·       копирование столбцов Excel-таблиц в табличную часть закладки «Исходная таблица»;

·       установка соответствия между реквизитами различных форм программного комплекса и колонками табличной части закладки «Исходная таблица» на закладке «Установка соответствий»;

·       проверка адекватности соответствия данных на закладке «Данные»;

·       автоматическая загрузка проверенных данных  при проведении документа «Загрузка долговых обязательств».

 

Форма документа состоит из:

·       рабочей области с реквизитами шапки документа, и трех закладок: «Данные», «Установка соответствий» и «Исходная таблица»;

·       области навигации, которая позволяет перейти к регистру накопления «Задолженность по объектам» и регистру сведений «События по объектам».

 

Заполнение закладок документа «Загрузка задолженности» также как и загрузка документа «Загрузка долговых обязательств» начинается с заполнения закладки «Исходная таблица», в которую путём копирования строк и столбцов из Excel-таблиц вносятся данные по задолженностям по различным долговым обязательствам. Следует особо обратить внимание на то, что первая строка табличной части данной закладки представляет собой заголовки колонок, соответствие которым будет устанавливаться на закладке «Установка соответствий».

 

Закладка «Установка соответствий» предназначена для указания Пользователем соответствия между графами табличной части закладки «Исходная таблица» и реквизитами различных форм объектов программного комплекса, перечисленными в виде граф табличной части закладки «Данные». Установка соответствий осуществляется в графе «Параметры связи» табличной части закладки. Для установки связи между необходимым свойством документа (реквизитом формы объекта программного комплекса) и колонками исходной таблицы в графе «Параметры связи» необходимо (в зависимости от вида информации в графе табличной части «Тип данных») выбрать соответствующее наименование поля из перечисления, сформированного из наименований соответствующих колонок табличной части закладки «Исходная таблица». В табличной части закладки указывается соответствие по следующим реквизитам:

·       Дата – дата возникновения задолженности (например, дата выставления счета контрагенту).

·       Объект – долговое обязательство контрагента, по которому производится начисление задолженности;

·       Валюта – валюта задолженности;

·       Основной долг – основная сумма задолженности контрагента по данному долговому обязательству;

·       Проценты – проценты по невыплаченной задолженности за период от даты возникновения задолженности по текущую дату;

·       Штрафы – общая фиксированная сумма штрафов за просрочку платежей по данному долговому обязательству.

В табличной части закладки «Данные» отражается вся информация о суммах задолженностей по долговым обязательствам, для которых определено соответствие на закладке «Установка соответствий». Для автоматического заполнения закладки предусмотрена кнопка «Заполнить таблицу» командной панели закладки.


После проведения документа «Загрузка задолженности» в справочнике «Долговые обязательства» обновляются данные по задолженностям в соответствии с данным документом. Загруженные суммы отражаются в соответствующих панелях дополнительного интерфейса форм. При проведении документа в регистре накопления «Задолженность по объектам» сумма задолженности отражается со знаком «+».
