Справочник "Шаблоны текста для автоинформирования"

 

Справочник «Шаблоны текста для автоинформирования» содержит шаблоны текста, используемые при формировании SMS сообщений и голосовых сообщений для автоинформирования должников.

 

Форма элемента справочника «Шаблоны текста для автоинформирования» состоит из реквизитов шапки формы, двух панелей («На примере» и «Шаблоны HTML»), а также табличных частей: «Прикрепляемые файлы» и «Параметры».

Вид формы и значение реквизитов формы зависит от значения реквизит типа «переключатель» "Назначение шаблона " шапки формы элемента справочника. В данном элементе справочника пользователь имееь указать значения данного реквизита: «SMS оповещение», «Голосовое сообщение» или «E-mail рассылка»

 

Ø SMS-оповещение;

Панель «На примере» включает в себя реквизит «ОбъектВх:», при помощи которого можно задать объект системы, для которого осуществляется выбор или создание данного шаблона автоинформирования. Выбор объекта осуществляется в два этапа:

o   На первом этапе из формы «Выбор типа данных» выбирается тип журнала объекта;

         o   На втором этапе из выбранного журнала объектов выбирается сам объект.

 

Заполнение реквизита «ОбъектВх» предназначено только для тестирования результатов адекватности выполнения функций данного шаблона. Тестирование шаблона автоинформирования может быть проведено только после создания самого шаблона при помощи кнопки «Заполнить шаблон», расположенной в командной строке справочника. При этом в поле «Пример текста» сформируется сообщение по данному шаблону для объекта, выбранного в панели «На примере».

В поле «Шаблон текста» создается шаблон текста для отправки SMS-сообщения, в которое включаются теги, выделенные значками <>. Значения тегов определяются в таблице «Параметры», расположенной в нижней части формы справочника. В поле «Количество символов» отражается число символов, введенных в поле «Шаблон текста», что наиболее актуально для SMS-сообщений.

Заполнение таблицы «Параметры» производится при помощи стандартных инструментов командной панели. Основные столбцы табличной части «Параметры»:

·        Поле – имя тега из текста шаблона сообщения для определения его значения;

·       Тип поля – вид данного параметра (выбирается из перечисления);

   ·       Значение – значение параметра.

o    Если в столбце «Тип поля» был выбран тип «Текст», то в столбце «Значение» данной табличной части необходимо указать текстовое значение подстановки;

o    Если в столбце «Тип поля» был выбран тип «Реквизит», то в столбце «Значение» данной табличной части необходимо выбрать реквизит (в т.ч. и дополнительный реквизит) различных форм при помощи формы окна «Выбор реквизитов».

o    Для «SMS оповещения» тип поля «Звуковой файл» не выбирается!!!

·       Функция – данная графа заполняется только при необходимости преобразования значения реквизита текстового шаблона, заданного в графе «Значение» (выбирается из справочника «Функции текстовых шаблонов» ).

 

Остальные реквизиты и табличные части не активны.

 

Ø Голосовое автоинформирование;

Общий вид и порядок заполнения элемента справочника «Шаблоны текста для автоинформирования» для варианта отправки голосового сообщения, аналогичен описанному варианту заполнения для отправки SMS-сообщения. Исключение составляет необходимость выбора в графе «Тип поля» табличной части «Параметры» значения «Звуковой файл», а в графе «Значение» – выбор шаблона файла голосового авоинформирования.

 

Ø Автоинформирование по электронной почте;

В случае необходимости создания шаблона текста для отправки mail-сообщения, элемент справочника видоизменяется. При этом:

·       становится активным реквизит «Шаблон HTML»;

·       становится активной табличная часть «Прикрепляемые файлы»;

·       табличная часть «Шаблон текста» заменяется на две табличные части;

·       и т.д.


В табличную часть «Прикрепляемые файлы» при помощи стандартных инструментов командной панели необходимо внести список всех дополнительных реквизитов, которые хранят в себе путь до прикрепленных файлов (графа «Свойство»).

Дополнительные реквизиты выбираются из раскрывающейся формы «Дополнительные реквизиты и сведения».

Реквизит «Шаблон HTML» формы элемента справочника «Шаблоны текста для автоинформирования» обязателен для заполнения при создании шаблона для е-mail сообщения. Выбор значений данного реквизита производится из справочника «Шаблоны HTML».

Две таблицы «Таблицы» и «Колонки таблицы», расположенные в средней части формы справочника служат для заполнения табличных частей шаблона. Добавление строк и редактирование таблиц осуществляется с помощью стандартных инструментов командной панели.

Количество строк в таблице «Таблицы» соответствует количеству табличных частей шаблона. После заполнения колонки «Функция» (выбирается из справочника «Функции текстовых шаблонов») в данной табличной части значениями выбранной функции, в таблице «Колонки таблицы» заполняются ячейки «Колонка». В колонке «Поле» для каждого значения функции указывается место в шаблоне, в которое она будет вставляться. Для этого копируется тег из шаблона текста.

Заполнение таблицы «Параметры» для элемента справочника «Шаблоны текста для автоинформирования» для отправки e-mail-сообщения аналогично заполнению этой таблицы для элемента справочника при отправке SMS-сообщения.

После заполнения всех реквизитов и таблиц при помощи кнопки «Заполнить шаблон», расположенной в командной панели элемента справочника (см. «SMS-оповещение»), заполняются поля:

·       Пример текста – пример текстового сообщения (заполняется для шаблонов при отправке SMS и голосовых сообщений):

   ·       Пример HTML – пример сообщения по электронной почте (заполняется для шаблонов при отправке e-mail сообщений).
