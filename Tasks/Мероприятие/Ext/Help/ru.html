Документ (задача) "Мероприятие"

 

Документ(задача) «Мероприятие» предназначен для отражения событий (действий) выполненных и запланированных для истребования задолженности по данному долговому обязательству.  Документы «Мероприятия» могут быть созданы как в процессе выполнения регламентных заданий, так и в ручном режиме.


Независимо от прав и роли Пользователя в организации форма документа является стандартной формой с реквизитами:

·       Организация – наименование организации, в которой (по умолчанию) работает сотрудник, создавший данный документ; значение данного реквизита может быть изменено Пользователем вручную – выбирается из элементов справочника «Организации»;

·       Подразделение – наименование подразделения организации, в которой (по умолчанию) работает сотрудник, создавший данный документ; значение данного реквизита может быть изменено Пользователем вручную – выбирается из элементов справочника «Подразделения организации»;

·       Ответственный – наименование Пользователя, ответственного за контроль выполнения данного мероприятия;

·       Исполнитель – наименование Пользователя, который должен выполнить данное запланированное мероприятие;

·       Автор – наименование Пользователя, создавшего данный документ;

·       Объект – наименование объекта (выбирается в два этапа; на первом этапе из открывающейся формы «Выбор типа данных» выбирается тип объекта, а затем из соответствующего справочника выбирается сам объект);

·      Тип мероприятия – наименование типа мероприятия (выбирается Пользователем вручную из элементов справочника «Типы мероприятия»).

 

 

В панели «Планирование» документа «Мероприятие» Пользователю необходимо указать планируемую дату выполнения данного мероприятия и астрономическое время его выполнения.  

 

После выполнения мероприятия в панели «Выполнение» Пользователь должен заполнить значения следующих реквизитов:

·       Фактическая дата – дата и время начала выполнения мероприятия;

·       Результат – результат проведённого мероприятия (выбирается из элементов справочника «Результаты мероприятия»);

·       Связанный документ – документ, связанный с выполнением данного мероприятия. Например, если при выполнении мероприятия «Телефонный звонок исходящий» создается документ «Обещанные платежи», то этот документ отражается в реквизите «Связанный документ» в форме документа «Мероприятие»;

·       Комментарий – текстовое поле, в которое при необходимости может быть внесена любая информация, касающаяся выполнения мероприятия (например, информация о выполнении данного мероприятия, причинах невозможности его выполнения или о характеристиках контактного лица должника).

 

В случае если в элементе справочника «Результаты мероприятия» присутствуют дополнительные реквизиты, то они также отражаются в панели «Выполнение». Поэтому Пользователю после выполнения мероприятия так же необходимо их заполнить. Например, для типа мероприятия «Получение информации о дате и месте судебного заседания» дополнительные реквизиты элемента справочника «Результаты мероприятия» могут являться: «Дата заседания» и «Место заседания», которые отражаются в панели «Выполнение».


Если при заполнении реквизита «Тип мероприятия» в элементе справочника «Типы мероприятия» задан реквизит «Функция для дополнительных данных», то в элементе справочника «Мероприятия» отразится дополнительное не редактируемое Пользователем поле «Информация», которое может содержать дополнительные данные, необходимые для выполнения мероприятия.

 

Кнопка «Вызвать сценарий диалога» командной панели формы служит для открытия элементов справочника «Сценарии диалога» и используется при работе с документом «Мероприятие» для типа мероприятия «Телефонный звонок исходящий».

 

При нажатии на элемент «Старт» в открывшемся элементе справочника «Сценарии диалога» из формы документа «Мероприятие», автоматически заполняются значения реквизитов панели «Планируемая дата» и значение реквизита «Фактическая дата» панели «Выполнение». После проведения беседы с должником по одному из вариантов сценария, представленного на блок-схеме, Пользователь двойным щелчком мыши выбирает один из конечных результатов разговора (прямоугольники, окрашенные на схеме голубым цветом). При этом в форме документа «Мероприятие» автоматически заполняется значение реквизита «Результат» панели «Выполнение».

 

Регистрация выполнения мероприятия осуществляется кнопками:

·      Выполнено - используется в случае, если после выполнения мероприятия необходимо оставить форму документа открытой;

·      Выполнено, закрыть - при использовании данной кнопки происходит регистрация выполнения данного мероприятия и закрытие формы документа.

 

 

После использования кнопок «Выполнено» или «Выполнено, закрыть» в панели «Выполнение» появляется отметка о выполнении данного мероприятия: «Выполнено!», окрашенная красным цветом.

   

В реквизит «Дата выполнения», расположенный в нижней части формы, после выполнения мероприятия Пользователю необходимо вручную ввести дату выполнения мероприятия.

 

Установка флажка в реквизит «Выполнена» производится автоматически в случае получения результата выполнения данного мероприятия.
