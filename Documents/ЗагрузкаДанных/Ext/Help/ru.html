Документ "Загрузка долговых обязательств"

 

Документ «Загрузка долговых обязательств» используется для автоматической загрузке (импорте) значительного количества объектов системы, например, долговых обязательств из Excel-таблиц. Принцип заполнения данного документа осуществляется в несколько последовательных этапов:

·       выгрузка всей необходимой информации из программного продукта Организации, в котором ведётся учёт долговых обязательств контрагентов, в Excel-таблицу;

·       копирование столбцов Excel-таблиц в табличную часть закладки «Исходная таблица» в которой заранее указаны наименования колонок соответствия, для которых далее устанавливается соответствие с наименованием реквизитов форм;

·       установка соответствия между реквизитами различных форм конфигурации (например, реквизитами элементов справочника «Долговые обязательства") и колонками табличной части закладки «Исходная таблица», осуществляемая на закладке «Установка соответствий»;

·       проверка адекватности соответствия данных, осуществляемая на закладке «Данные» при помощи кнопки командной панели закладки «Заполнить таблицу»;

·       автоматическая загрузка проверенных данных в программном комплексемпри проведении документа «Загрузка долговых обязательств».

 

Форма документа «Загрузка долговых обязательств» состоит из:

·       рабочей области с реквизитами шапки документа и трёх закладок: «Данные», «Установка соответствий» и «Исходная таблица»;

·       области навигации, которая позволяет перейти к регистру сведений «Сообщения по загрузке данных».


При заполнении шапки формы особое внимание обратить на реквизиты:

·       Тип контрагентов – классификационный тип контрагента. Выбирается из перечисления, сформированного из элементов справочника «Типы контрагентов»;

  ·      Тип долговых обязательств – классификационный тип долговых обязательств. Выбирается из перечисления, сформированного из элементов справочника;


Заполнение закладок документа начинается с заполнения закладки «Исходная таблица», в которую путём копирования строк и столбцов вносятся данные по долговым обязательствам из Excel-таблиц. Следует особо обратить внимание на то, что первая строка табличной части данной закладки представляет собой заголовки колонок, соответствие которым будет устанавливаться на закладке «Установка соответствий»!

 

Закладка «Установка соответствий» предназначена для указания Пользователем соответствия между графами табличной части закладки «Исходная таблица» и реквизитами различных форм объектов программного комплекса перечисленными в виде граф табличной части закладки «Данные». Установка соответствий осуществляется в графе «Параметры связи» табличной части закладки "Данные". Для установки связи между необходимым свойством документа (реквизитом формы объекта программного комплекса) и колонками исходной таблицы в графе «Параметры связи» необходимо (в зависимости от вида информации в графе табличной части «Тип данных») выбрать соответствующее наименование поля из перечисления, сформированного из наименований соответствующих колонок табличной части закладки «Исходная таблица».

 

В табличной части закладки «Данные» отражается вся информация по долговым обязательствам, для которой определено соответствие на закладке «Установка соответствий». Для автоматического заполнения закладки предусмотрена кнопка «Заполнить таблицу» командной панели закладки.

 

В нижней части закладки расположены следующие реквизиты:

·       Поставить в очередь – при установке флажка в данном реквизите задание на внесение данных по данному документу будет поставлено в очередь для выполнения регламентных заданий (см. п. 7.1.3);

·       Обновлять ранее записанные – при установке флажка в данном реквизите при повторном проведении документа все внесённые ранее сведения по данным долговым обязательствам автоматически заменяются на вновь заданные.

 

После проведения данного документа (в зависимости от наличия флажка в реквизите «Обновлять ранее записанные») в справочниках «Контрагенты»  и «Долговые обязательства» автоматически создаются новые элементы или в данных элементах справочника обновляется вся ранее внесённая информация.
