Объект "Бизнес процесс"

 

Объект «Бизнес-процесс» предназначен для выполнения стадий БП по объекту (контрагенты, договоры контрагентов, долговые обязательства, услуги по договору или исполнительные документы). Все объекты данного типа хранятся в журнале объектов «Бизнес-процессы»:

Объекты «Бизнес-процесс» могут существовать в виде отдельной независимой линейки действий, так и запускаться в виде вспомогательных блок-схем действий в процессе выполнения бизнес-процесса «родителя». 

 

Форма объекта «Бизнес-процесс» состоит из:

·       реквизитов шапки формы;

·       поля графической схемы, содержащей блок схему бизнес-процесса;

·       табличной части «Стадии процесса»;

·       табличной части «Мероприятия».

 

При заполнении шапки формы объекта особое внимание обратить на реквизиты:

·       Объект - цель выполнения действий данного бизнес-процесса (недоступна Пользователю для внесения информации в программный комплекс, т.к. привязана к схеме выполнения бизнес процесса, и после выбора схемы БП имеет другой вид);

·      Схема – последовательность выполнения отдельных блоков (точек БП) каждого бизнес-процесса, который будет запущен данным документом (выбирается из элементов справочника «Карты маршрутов»);

·       Процедура – наименование группы операций (действий) бизнес-процессов (выбирается из элементов справочника «Процедуры Бизнес-процессов»);

·       Стартован – установка флажка в данном реквизите отражает начало работ по выполнению задач данного бизнес-процесса (устанавливается автоматически при старте бизнес-процесса - блок «Старт» поля графической схемы);

·       Завершён – установка флажка в данном реквизите отражает завершение работ по выполнению задач данного бизнес-процесса (устанавливается автоматически при логическом завершении алгоритма карты маршрута);

·       Результат – текстовое поле с результатом выполнения БП.

 

В табличной части «Стадии» указывается наименование выполненных ранее и текущей применяемой схемы (карты маршрута) выполняемого бизнес-процесса.

В табличной части «Мероприятия» отражается весь список мероприятий, выполненных на всех стадиях истребования задолженности по данному долговому обязательству. Для каждой строки, отражающей отдельное мероприятие, заполняются следующие графы:

·       Планируемая дата – предполагаемая дата выполнения данного мероприятия;

·      Тип мероприятия - вид запланированного мероприятия (см. п. 5.1.1), согласно действующему в Организации порядку истребования задолженности;

·       Номер и Дата создания - номер и дата формирования соответствующего документа «Мероприятие» (см. п. 4.2.3);

·       Результат - эффективность выполнения мероприятия (см. п. 5.1.2);

·       Фактическая дата - истинная дата выполнения мероприятия;

·      Схема - наименование карты маршрута использованного бизнес-процесса, в рамках которого выполнено данное мероприятие.
